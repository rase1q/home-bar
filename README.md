# Домашний бар

Публичное mobile-first меню домашнего бара. Интерфейс бесплатно размещается на GitHub Pages, данные, вход администратора и фотографии хранятся в Supabase.

## Возможности

- подбор коктейлей по сладости, кислоте, крепости, основе и поиску;
- карточка рецепта с пропорциями, способом приготовления, бокалом и украшением;
- автоматическое исключение рецептов с отсутствующими ингредиентами;
- защищённая панель администратора;
- стоп-лист с поиском и массовым переключением;
- добавление, переименование и удаление основ для коктейлей;
- создание, редактирование и удаление коктейлей;
- редактирование рецептов коктейлей в админке;
- загрузка JPG, PNG и WEBP до 5 МБ;
- адаптивный интерфейс для телефонов и планшетов.

## Первичная настройка Supabase

1. Откройте **SQL Editor** своего проекта Supabase.
2. Скопируйте и выполните целиком файл [`supabase/schema.sql`](supabase/schema.sql). Он создаст таблицы, RLS-политики, хранилище фотографий и стартовые рецепты.
3. Откройте **Authentication → Users → Add user → Create new user**.
4. Укажите email `homebar-admin@example.com`, придумайте пароль не короче 8 символов и включите подтверждение пользователя.
5. В **Authentication → Sign In / Providers → Email** отключите публичную регистрацию (**Allow new users to sign up**), поскольку администратор уже создан.

В интерфейсе вводится только пароль: технический email администратора зафиксирован в приложении. Публичный Supabase key не является секретом; безопасность обеспечивают RLS-политики, разрешающие запись только указанному email.

## Обновление существующего проекта

Если база уже была создана раньше, выполните один раз файл [`supabase/add-cocktail-recipes.sql`](supabase/add-cocktail-recipes.sql) в **Supabase → SQL Editor**. Он добавит таблицу редактируемых рецептов и не перезапишет уже изменённые рецепты.

## GitHub Pages

Workflow `.github/workflows/pages.yml` публикует папку `public` после каждого push в `main`.

В репозитории откройте **Settings → Pages → Build and deployment → Source** и выберите **GitHub Actions**. После выполнения workflow сайт будет доступен по адресу:

`https://rase1q.github.io/home-bar/`

## Локальный просмотр

Раздайте папку `public` любым статическим сервером, например:

```bash
npx serve public
```
